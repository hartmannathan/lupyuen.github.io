                                üìù 11 Nov 2021

                                Previously we tested two new wireless gadgets by Pine64‚Ä¶

                                • PineDio Stack BL604 RISC-V Board

                                • PineDio LoRa USB Adapter

                                Both gadgets transmit and receive small data packets over incredible distances thanks to LoRa, the Long-Range Low-Bandwidth wireless network.

                                (Up to 5 km or 3 miles in urban areas‚Ä¶ 15 km or 10 miles in rural areas!)

                                Today we test the third LoRa gadget by Pine64: PineDio LoRa Gateway

                                What‚Äôs a LoRa Gateway? How does it differ from other LoRa gadgets?

                                PineDio Stack and PineDio USB are perfectly fine for Point-to-Point Wireless Communication.

                                But if we need to relay data packets to multiple devices or to the internet, we need a LoRa Gateway like PineDio Gateway.

                                (It‚Äôs like a WiFi Router, but for LoRa)

                                LoRa works over the internet?

                                Yes when we connect PineDio Gateway to The Things Network, the free-to-use public global network for LoRa gadgets.

                                (We‚Äôll learn how in a while)

                                The Things Network is a public LoRa network. Why do we need PineDio Gateway?

                                Network Coverage for The Things Network is spotty in some regions.

                                Hopefully Pine64 will make PineDio Gateway highly affordable for Schools, Workplaces and Homes to install everywhere‚Ä¶ And grow The Things Network!

                                (Coverage map for The Things Network)

                                The data packets received by The Things Network‚Ä¶ What can we do with them?

                                We may configure The Things Network to‚Ä¶

                                • Push the packets to an MQTT Queue

                                • Store now and fetch later over HTTP

                                • Trigger an IFTTT Action

                                • Forward to another LoRa Device

                                • And do much more! (See this)

                                What about other LoRa networks?

                                PineDio Gateway runs an open source LoRa Network Stack. (Based on Arm64 Linux)

                                We could possibly integrate PineDio Gateway with other LoRa Mesh Networks‚Ä¶ Like Meshtastic (Data Mesh), QMesh (Voice Mesh) and Mycelium Mesh (Text Mesh).

                                Will PineDio Gateway support Helium Network?

                                Probably not. Our pre-production PineDio Gateway doesn‚Äôt have a Cryptographic Co-Processor.

                                (More about Cryptographic Co-Processors)

                                Can we run our LoRa Gateway on a local network, without connecting to the cloud?

                                Yep just install ChirpStack, the open source LoRaWAN Stack.

                                (LoRa Packets received by ChirpStack will be published to a Local MQTT Queue)

                                (Source)

                                1 Inside PineDio Gateway

                                What‚Äôs inside PineDio Gateway?

                                Our pre-production PineDio Gateway has two boards inside‚Ä¶

                                • PINE A64-LTS Arm64 single-board computer

                                • RAKwireless RAK2287 LoRa Module with Semtech SX1302 Concentrator

                                What‚Äôs a LoRa Concentrator? How does it differ from a LoRa Transceiver?

                                LoRa Transceivers (like SX1262 in PineDio Stack and PineDio USB) are designed to talk to one LoRa device at a time.

                                LoRa Concentrators (like SX1302 in PineDio Gateway) can handle data packets from multiple LoRa devices across multiple frequencies at the same time.

                                That‚Äôs why LoRa Gateways have a LoRa Concentrator inside.

                                (And nope, we can‚Äôt build a proper LoRa Gateway with a plain LoRa Transceiver)

                                What ports and connectors are on PineDio Gateway?

                                In the pic above we see connectors for‚Ä¶

                                • GPS Antenna

                                • LoRa Antenna

                                • HDMI Output

                                • Ethernet (10 / 100 Mbps)

                                • DC Power (5V)

                                (All these need to be connected except HDMI, which is useful for troubleshooting)

                                The connectors not shown are microSD, USB 2.0, Audio Input / Output.

                                Note that we‚Äôre testing the pre-production PineDio Gateway, so some features may change‚Ä¶

                                Which is super awesome because it means we have (some) Wireless Redundancy in The Things Network!

                                5 Benchmark with RAKwireless WisGate

                                How does PineDio Gateway compare with other LoRa Gateways?

                                Let‚Äôs benchmark PineDio Gateway with RAKwireless WisGate D4H Gateway (RAK7248). (Pic above)

                                WisGate D4H is based on the same RAKwireless RAK2287 + Semtech SX1302 Concentrator as PineDio Gateway.

                                Thus we would expect the wireless performance of the two LoRa Gateways to be highly similar.

                                5.1 Compare LoRa Reception

                                How shall we benchmark the two gateways for receiving LoRa packets?

                                We log on to The Things Network, browse to the two Gateways and view the Live Data‚Ä¶

                                For a single received packet, we see that the Received Signal Strength (RSSI) recorded by PineDio Gateway is slightly weaker than WisGate D4H‚Ä¶

                                • PineDio Gateway: -108 dBm

                                • WisGate D4H: -103 dBm

                                (Higher numbers are better‚Ä¶ -103 is better than -108)

                                Across multiple packets (pic above), we see that the Received Signal Strength recorded by PineDio Gateway (left) is generally slightly weaker than WisGate D4H (right)‚Ä¶

                                • PineDio Gateway: -106 dBm to -110 dBm

                                • WisGate D4H: -101 dBm to -105 dBm

                                This suggests that WisGate D4H might receive slightly more packets than PineDio Gateway. Especially if the packets were transmitted far from the Gateway.

                                (We‚Äôre talking packets in the -110 dBm to -120 dBm range‚Ä¶ Close to the reception limit of LoRa Gateways)

                                What about packets transmitted from nearby devices?

                                For a packet transmitted near the Gateways (pic above), the Received Signal Strength for PineDio Gateway is still slightly weaker than WisGate D4H‚Ä¶

                                • PineDio Gateway: -57 dBm

                                • WisGate D4H: -52 dBm

                                But this shouldn‚Äôt be a problem‚Ä¶ Due to the higher Signal Strength, PineDio Gateway will receive the same packets as WisGate D4H.

                                WisGate Antenna (left) vs PineDio Gateway Antenna (right)

                                5.2 Compare Antennas

                                Both LoRa Gateways are based on the same LoRa Concentrator. Why the difference in LoRa Reception?

                                The WisGate and PineDio Gateways have different antennas.

                                This might affect the LoRa Reception for the gateways.

                                Unfortunately we can‚Äôt swap the two antennas and test‚Ä¶ The Antenna Connectors don‚Äôt match. (Pic above)

                                I see a pattern‚Ä¶ Why is one antenna twice the length of the other?

                                That‚Äôs because LoRa Antennas are typically ‚ÄúŒª / 2‚Äù or ‚ÄúŒª / 4‚Äù long.

                                Let‚Äôs do the math‚Ä¶

                                • In my region the LoRa Frequency is 923 MHz

                                • Which means Wavelength (Œª) is 32 cm (rounded)

                                  (Source)

                                • ‚ÄúŒª / 2‚Äù is 16 cm

                                • ‚ÄúŒª / 4‚Äù is 8 cm

                                Which matches our Antenna Lengths!
