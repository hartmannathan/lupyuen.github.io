                ¬ß5 Why So Difficult?

                1. What‚Äôs this Pytest we saw earlier?

                  Filipe Cavalcanti wrote an excellent article on Pytest in NuttX‚Ä¶

                  ‚ÄúTesting applications with Pytest and NuttX‚Äù

                  Pytest is a Python Testing Framework. Though it gets messy because NuttX CI isn‚Äôt actually testing Python Code with Pytest‚Ä¶

                  It‚Äôs testing External Programs in QEMU. (Because NuttX boots inside QEMU)

                2. How does Pytest control QEMU?

                  Remember test_helloxx that failed earlier? It calls‚Ä¶

                  • send_command to send an NSH Command, which calls‚Ä¶

                  • Pexpect to talk to QEMU

                3. Why Pexpect?

                  That‚Äôs how our CI Test spawns the QEMU Process and controls it‚Ä¶

                  ‚ÄúPexpect is a pure Python module for spawning child applications; controlling them; and responding to expected patterns in their output. Pexpect works like Don Libes‚Äô Expect. Pexpect allows your script to spawn a child application and control it as if a human were typing commands.‚Äù

                4. What about Don Libes‚Äô Expect?

                  Oh yes we use Plain Old Expect for Daily-Testing SG2000 NuttX and Ox64 NuttX. It terminates reliably, and it won‚Äôt hang forever, unlike Pexpect.

                  (Hence this Horrible Hack)

                5. Why so hard to extract the CI Test Log?

                  Yeah we should probably add the CI Test Log to GitHub Actions Artifacts for easier downloading. Then NuttX Dashboard can ingest the CI Test Log and render it sensibly.

                  (Also: Extract the Leftover Files that fail the defconfig check)

                6. Can we do more with Pytest?

                  Someday we could call Pytest to do Test-Driven Development of NuttX Apps and NuttX Drivers. Which means we can write the Test Cases in Pytest, before writing a Single Line of NuttX Code!

                No more CI Test errors yay!
